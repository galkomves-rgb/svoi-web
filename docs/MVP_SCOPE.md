# MVP Scope

## Included in Current MVP Baseline
- responsive app shell with header, navigation, homepage, and footer
- geo-first routing through `/[city]/...`
- mock data layer for cities, listings, events, services, and guides
- working city dashboards
- working listings browse flow with detail pages
- events, services, and guide pages
- start page for newcomers
- initial documentation for product and technical direction
- base UI patterns for cards, lists, tags, and empty states

## Excluded from MVP
- real authentication
- real backend or database
- real-time chat
- real dating chat implementation
- controlled photo reveal with persistence
- advanced real estate filters
- listing submission forms with persistence
- payments and monetization flows
- moderation dashboard implementation

## Current Runtime Status
- Implemented now:
- Start
- Listings
- Events

- Guide
- Dating foundation

- Planned next:
- Real Estate

## Launch Order by Module
1. Listings
2. Events
3. Services
4. Start
5. Dating
6. Community Resources
7. Real Estate

## Why This Launch Order Is Correct
- Listings already carry the clearest practical value and form the current product core.
- Events add social proof and recurring local reasons to return.
- Services are the clearest early monetization surface after listings.
- Start should become a strong onboarding layer that connects newcomers with the core modules.
- Dating should launch as a narrow privacy-first foundation, not as a full chat product.
- Community Resources should follow once the route strategy and geo model are stable.
- Real Estate should come later because users expect better structure, richer content, and stronger trust signals.
